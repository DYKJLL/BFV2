# V1.2.4 修复说明

## 修复日期
2026 年 4 月 15 日

## 修复内容

### 首页按钮逻辑修正

**问题描述：**
- 剧迷模式下，用户在看片狂人浏览视频
- 点击首页按钮后，回到了第一个站点"片片"（错误！跨站点了）
- 用户期望：应该回到看片狂人的首页（同一个站点内）

**根本原因：**
- 首页按钮依赖 `quickDramaSelect.value` 判断首页
- 但用户可能通过侧边栏按钮进入站点，下拉框没有同步
- 导致下拉框值和实际页面不匹配

**修复方案：**
1. 首页按钮改为**根据地址栏当前 URL 判断用户在哪个站点**
2. 匹配 `dramaSites` 找到对应的站点
3. 返回这个站点的首页（不会跨站点）
4. 侧边栏按钮点击时同步下拉框状态

**代码位置：**
- `assets/js/renderer.js` line 394-428 (首页按钮)
- `assets/js/renderer.js` line 757-772 (侧边栏按钮)

**修复前代码：**
```javascript
// ❌ 问题：依赖下拉框的值，可能和实际页面不匹配
homeButton.addEventListener('click', () => {
    const homeUrl = quickDramaSelect.value || dramaSites[0].value;
    window.voidAPI.resetModule(homeUrl);
});
```

**修复后代码：**
```javascript
// ✅ 修复：根据地址栏当前 URL 判断实际在哪个站点
homeButton.addEventListener('click', () => {
    const currentUrl = urlInput.value.trim();
    const currentSite = dramaSites.find(site => currentUrl.startsWith(site.value));
    const homeUrl = currentSite ? currentSite.value : (quickDramaSelect.value || dramaSites[0].value);
    
    quickDramaSelect.value = homeUrl;
    urlInput.value = homeUrl;
    loadingOverlay.classList.remove('hidden');
    window.voidAPI.resetModule(homeUrl);
});
```

**效果对比：**

| 修复前 | 修复后 |
|--------|--------|
| 用户在"看片狂人"点首页 | 回到第一个站点"片片" ❌ |
| 用户在"看片狂人"点首页 | 回到"看片狂人" ✅ |
| 依赖下拉框值（可能不匹配） | 根据实际页面 URL 判断 |
| 可能跨站点 | 必定在同一站点内 |

**用户场景示例：**
```
场景：剧迷模式下，用户通过侧边栏进入"看片狂人"浏览视频

修复前:
  1. 用户在看片狂人
  2. 点击首页 🏠
  3. ❌ 回到"片片"（第一个站点，跨站点了）
  4. 用户懵逼：我在看片狂人，怎么跳到片片了？
  
修复后:
  1. 用户在看片狂人
  2. 点击首页 🏠
  3. ✅ 回到"看片狂人"首页（同一个站点）
  4. 用户满意：对了，这才是我要的！
```

## 打包方式

开发完成后，在本地打包：

```bash
# 安装依赖
npm install

# Windows 打包
npm run dist:win

# macOS 打包
npm run dist:mac

# Linux 打包
npm run dist:linux

# 全平台打包
npm run dist:all
```

打包完成后，安装包在 `dist/AudioVisual-1.2.4-x64.exe`，约 81 MB。
